## How to Use

Each document covers:

1. **Overview** — what the framework is and who publishes it
2. **Scope** — what it applies to (org, people, tech, process)
3. **Key Controls / Requirements** — numbered requirements or control families
4. **Certification** — what certification looks like (if any)
5. **Relevance** — when you need this framework
6. **Relationship** — how it maps to other frameworks
